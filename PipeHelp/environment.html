Environment Variables

This is a list of all environment variables provided by Pipe for your scripts.

PIPE_INPUT_CHARSET
PIPE_EXPECTED_OUTPUT_CHARSET
PIPE_SCRIPT_CHARSET
The IANA (internet) names of the charsets used for the script source code, input, and the expected output (e.g. UTF-8).
PIPE_EXPECTED_OUTPUT_TYPE
Contains either text, image or html depending on the currently selected output type.
PIPE_SCRIPT_SUPPORT_PATH
The path to the Support Files directory inside the current script bundle. Note that this variable is only set if the current script was saved to disk at least once Ð it is not available to unsaved scripts.
PIPE_SERVICE_REQUESTING_APP_BUNDLE_IDENTIFIER
PIPE_SERVICE_REQUESTING_APP_BUNDLE_NAME
PIPE_SERVICE_REQUESTING_APP_BUNDLE_PATH
The bundle identifier, bundle path, and name of the application that is currently requesting a service from Pipe. If a script is not run as a result of a service invocation, these variables are unset.
PIPE_SUPPORT_PATH
The path to the directory which contains various script support files bundled with Pipe.
PIPE_TEMP_PATH
A writable path for temporary working files. Pipe guarantees that this path is initially empty upon script launch, and that all files in this directory will be deleted when the script terminates.
PIPE_VERSION
The version number of Pipe (e.g. 130 for version 1.3).

Note: Your script should not make assumptions about the location of paths specified in the environment variables above, not even the relative locations to each other. Additionally, your script should not make assumptions about the initial working directory.
